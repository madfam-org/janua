## â“ Frequently Asked Questions

### "Why is SSO free? Everyone else charges for it."

Because **we believe it's wrong to charge for essential security features**. Enterprise SSO is table stakes for B2B SaaS. Making it a paid add-on is a tax on companies trying to secure their users.

We charge for making SSO **easy** (zero-config, automatic setup), not for the **feature itself**.

### "How do you make money if everything is free?"

1. **Plinto Cloud** (managed hosting convenience)
2. **Enterprise Services** (setup assistance, migration help, training)
3. **Compliance Packaging** (SOC 2, HIPAA certifications access)
4. **SLA Guarantees** (99.99% uptime, dedicated support)

We make money from **service and convenience**, not **feature access**.

### "Can I really use Plinto OSS for production?"

**Yes.** It's the same code that powers Plinto Cloud. MIT license. No restrictions. No hidden fees.

You'll need to:
- Manage your own infrastructure (Docker/K8s)
- Handle backups and monitoring
- Set up SSL certificates
- Perform version upgrades

If you have DevOps capacity, OSS is a great choice.

### "What if I outgrow the Starter plan?"

Upgrade to Professional or Scale. No migration needed. Just change billing.

Pricing is **transparent and predictable**: Based on MAU, not on "contact sales."

### "What happens if I stop paying?"

**Plinto Cloud**: Your hosted instance stops. But you can:
1. Export all your data (API available even after cancellation)
2. Deploy Plinto OSS on your infrastructure
3. Import your data and point DNS to your server
4. Continue running with zero data loss

**OSS**: Nothing. You own it. Keep running it forever.

### "Do I own my data?"

**Yes. Always.**

- OSS: Data is in YOUR database that YOU control
- Cloud: Data is in OUR database, but YOU can export it anytime
- Enterprise: Data can be in YOUR VPC if needed

We never "own" your users. You do.

### "Can I modify Plinto for my needs?"

**OSS**: Yes. MIT license. Fork it, modify it, sell it.  
**Cloud**: No. Managed service, can't modify core.  
**Enterprise**: Custom features available (we build for you).

### "What's the catch?"

**There is no catch.**

We genuinely believe authentication should be free and open. We're building a sustainable business by charging for **service**, not **features**.

If you prefer self-hosting, use OSS forever for $0. We're happy.  
If you want convenience, pay for Cloud. We're happy.  
If you need enterprise services, pay for Enterprise. We're happy.

**You choose your path. We don't trap you.**
